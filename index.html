MSc Research

Solving cold starts with ML.

Low-Latency Cold Start Optimisation for Serverless Machine Learning Inference

Cold starts are the silent latency killer of serverless ML — when a function hasn't been called recently, the cloud provider needs to spin up a new container, load your model, and then serve the request. This thesis builds a platform-agnostic ML framework that predicts when cold starts are likely to occur and pre-warms functions to prevent them — across AWS Lambda, Azure Functions, and Google Cloud Run simultaneously.

The Problem

Serverless ML inference suffers from cold start latency — delays of seconds when containers spin up from cold. Existing solutions are platform-specific or require manual tuning. No ML-driven, cross-cloud solution existed.

The Approach

Built a monitoring VM that collected cold start data across all three platforms simultaneously, loading real ML models (ResNet50, DistilBERT, Random Forest). Trained a Logistic Regression classifier with cost-aware threshold optimisation (threshold: 0.15) to predict cold start risk and trigger pre-warming.

Why It Matters

Multi-cloud ML deployments are increasingly common, but tooling is still platform-silo'd. A single framework that works across AWS, Azure, and GCP — and uses ML to adapt dynamically — fills a real gap in the serverless infrastructure space.
